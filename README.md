RESTful
: REST의 구현원칙을 제대로 지키는 시스템
 -- RESTful의 제약 조건
    - 클라이언트 -서버 :
        관심아의 명확한 분리 [관심사의 명확한 분리 -> 서버의 구셩요소가 단순화되고 확장성이 향상 -> 여러 플랫폼 지원 ]
    - 무상태성 :
        서버에 클라이언트의 상태 정보를 저장하지 않는 것 
            단순히 들어오는 요청만 처리하여 구현을 더 단순화
    - 캐시 기능 :
        REST는 HTTP의 장점을 그대로 계승 -> 클라이언트의 응답을 캐시할 수 있다.
    - 계층화 시스템 :
        서버는 중개 서버(게이트웨이, 프록시)나 로드 밸런싱, 공유 캐시 등의 기능을 사용하여 확장성 있는 시스템을 구성
    - 코드 온 디 맨드 :
        클라이언트는 서버에서 자바 애플릿, 자바스크립트 실행 코드를 전송받아 기능을 일시적으로 확장 가능
    - 인터페이스 일관성 :
        URI로 지정된 리소스에 균일하고 통일된 인터페이스를 제공 -> 아키텍처를 단순하게 분리하여 독립적으로 만듦
            - 세부 원칙 :
                - 자원 식별
                - 메시지를 통한 리소스 조작
                - 자기 서술적 메시지
                - 애플리케이션 상태에 대한 엔진으로서의 하이퍼미디어
                    - 클라이언트에 응답할 때, 단순히 결과 데이터만 제공해주기보다는 URI를 함게 제공해야 한다.
